Get started

This is the API documentation for the a Form Submission System, as shown on this github. It does not validate any input, in terms of what the message should contain, but does filter out spam.

To use this API for creating collections of forms, you need an API key. You can submit forms without an API key, of course. This can be done using JSON objects as a request body

How to submit a form

To submit a form, you make a POST call to: (Replace value in [  ] )
https://formapi.example.com/forms/[form_name]/submit

The following is an example how a form request could look like with parameters, but this is entirely free for implementation.

Example

Field Type Value
name String Koen de Jong
email String mail@koendejong.net
gender String female
message String Feedback is always appreciated
subscription Boolean true

In JSON format, it would look like this:

  {
    "name": "Koen de Jong",
    "email": "mail@koendejong.net",
    "gender": "female",
    "message": "Feedback is always appreciated",
    "subscription": true,
  }

Error handling

The Form Submission System API does not have error codes, but it does have error states and messages.

Every request will return a success boolean value, if either an error occurred or not, and a message string value, stating either an explaination of the error or that what the request has done.

Examples of possible returned objects

success message
true Form data inserted successfully
true Form Collection created successfully
false Form does not exist
false There was an database connection error. Check the logs for more info.
false No API key provided
false Invalid API key
false Form already exists
