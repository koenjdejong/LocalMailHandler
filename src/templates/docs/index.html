Get started

This is the API documentation for a LocalMailHandler, as shown on this github repo. It does not validate any input, in terms of what the message should contain.

To use this API for sending emails to different receivers, you need an API key. You can submit forms without an API key, of course. This can be done using JSON objects as a request body

How to submit a form

To submit a form, you make a POST call to:
https://forms.example.com/form/submit/

The following is an example how a form request could look like with parameters, but this is entirely free for implementation.

No API key is necessary for this.

Example

Field Type Value
name String Koen de Jong
email String mail@koendejong.net
gender String male
message String Feedback is always appreciated
subscription Boolean true

In JSON format, it would look like this:

  {
    "name": "Koen de Jong",
    "email": "mail@koendejong.net",
    "gender": "female",
    "message": "Feedback is always appreciated",
    "subscription": true,
  }

Sending emails

To be able to sent emails to different receivers than the one specified, you make a POST call to:
https://forms.example.com/mail/send/

To be able to send emails, you must specify a valid API key in a header. The table shows an example.

Key Value
API-KEY IBo3a9xkcFDY2H7Jv75h3k78OvjHlEee

You should also send data along with your request.

Example

Field Type Value
receiver String mail@koendejong.net
subject String Confirmation of Enrollment
body String Dear user, you have been successfully enrolled in our program. Kind regards, A lovely traveler

In JSON format, it would look like this:

  {
    "receiver": "mail@koendejong.net",
    "subject": "Confirmation of Enrollment",
    "gender": "Dear user, you have been successfully enrolled in our program. Kind regards, A lovely traveler",
  }

Error handling

The LocalMailHandler API does not have error codes, but it does have error states and messages.

Every request will return a success boolean value, if either an error occurred or not, and a message string value, stating either an explanation of the error or that what the request has done.

Examples of possible returned objects

success error code message
true 200 Form submitted successfully.
true 200 Email sent successfully.
false 400 No data could be found
false 400 Missing requirement body
false 401 Unauthorized. Invalid API key
false 401 Missing API key. A header 'API-KEY' should be present.
false 500 Internal server error. Check the logs for more info.
